## Fix list (decided design, implement as specified)

**F1 — engine hardening (P1/P2/C1/C3/S2/S4)** [engine agent]
- `end_turn()` / `notify_reply_sent()` must NEVER raise into a
  responder turn: wrap bodies; on failure log error (ls style, Activity
  tag) and leave safe state. (Live incident class: one unguarded await
  killed the run TaskGroup.)
- `_run_return` reorder for idempotency + no-stuck-absent:
  1. generate experience (already guarded w/ fallback)
  2. `finish_activity` FIRST — commits the return; restart can no
     longer replay the whole flow (no duplicate facts/turns), and any
     later failure leaves her BACK, not stuck absent
  3. then individually-guarded best-effort steps: event-fact, return
     turn, archive watermark, staged promotion (F3), missed-ping scan,
     wake publish, presence. One step failing must not kill the rest;
     log each failure.
  - `finally`: clear `_active`/`_returning`/departure state.
- `_sleep_then_return`: catch-all except → log exception (no orphan
  task exceptions, no silent stuck-absent).
- C1 race: clear `_active` (and set state such that `gate()` returns
  NORMAL) BEFORE publishing the return wake. Also: `gate()` returns
  NORMAL while `self._returning` — the wake and any message arriving
  mid-return must not be suppressed by a corpse state.
- C3 boot recovery: `start()` must NOT run a past-due return inline.
  Arm the normal return timer with a floor delay (~20s) so the bus
  consumers + Discord login exist before the wake publishes. Past-due
  ⇒ fire at now+floor.
- S2 judgment latch: one judgment turn per author per absence. After a
  JUDGMENT decision for author key X, further real pings from X during
  the same absence ⇒ SUPPRESS (the responder's existing suppressed+
  pings_bot path notes them for the return wake). Latch clears at
  activity start. Key on the payload author's user id (verify field).
- S4 nit: truncate the author display name interpolated into the
  state line (~40 chars).

**F2 — gate-vs-focus precedence (C2)** [engine agent]
- `gate()` returns JUDGMENT only when the event's channel == focused
  text channel (engine has `self._focus.get_focus("text")`; payload
  carries the channel id — verify key name in responder/source).
  Unfocused real ping ⇒ SUPPRESS: the responder then stages it AND
  calls `note_missed_ping` (verify that path: suppressed branch runs
  before the unfocused-return — it does today).
- Rewrite `test_cross_channel_ping_reaches_wake` to go through a real
  TextResponder (no hand-calling note_missed_ping) — the audit called
  the current test dishonest. Also add the missing responder-in-loop
  test: engine publishes its return wake onto a bus a REAL responder
  (fake LLM) consumes while the engine is mid-return — wake must
  produce a normal turn, not SUPPRESS (pins C1).

**F3 — staged promotion at return (C4)** [store agent, then engine]
- store: new `promote_staged_turns_since(*, familiar_id,
  after_turn_id) -> int` — set `consumed_at = now` for ALL channels'
  turns with `consumed_at IS NULL AND id > after_turn_id`. Returns
  rowcount. Mirrors existing promote_staged_turns style (find it —
  FocusManager uses it). Scoping to `id > departure` is deliberate:
  pre-absence staged turns in never-attended channels keep their
  attentional semantics.
- engine `_run_return`: call it (guarded) with the departure turn id —
  "she reads the screen when she gets back." Tests: chatter during
  absence (no pings) is consumed after return and appears in
  `recent_cross_channel`.

**F4 — alarm passthrough (P3)** [waker agent]
- AlarmWaker's synthetic discord.text payload gains a marker key
  `alarm: True` (waker.py — match existing payload shape).
- engine `gate()`: payload with `alarm is True` ⇒ NORMAL always (her
  own alarm pierces any absence; she responds and remains out —
  replying to a NORMAL turn does not cut short). [engine agent]
- Tests both sides: waker payload carries the key; gate passes alarms
  through while absent (reachable AND unreachable types).

**F5 — experience turn role (S1)** [engine agent]
- Return turn `role="system"` → `role="assistant"` (her own narration;
  kills the untrusted-text-at-system-authority replay). mode tag +
  content prefix unchanged. Update tests asserting role.

**F6 — doc drift** [docs agent]
docs/architecture/activities.md corrections to match post-fix reality:
- cross-channel pings: suppressed+noted (not judged), surface in the
  return wake (after F2 they really do)
- judgment turns: one per author per absence (latch)
- staged messages during absence are PROMOTED at return — window shows
  them; "history stays complete" now true for the window too
- return turn is assistant-role; lands at current focus at return
- restart recovery: floor-delay timer, wake survives restart; alarms
  pierce absences
- data/familiars/_default/activities.toml comment: min_gap_minutes
  gates NUDGES only (not tool starts)
